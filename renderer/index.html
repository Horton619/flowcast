HOW TO USE FLOWCAST

What is FlowCast?

FlowCast is a live-event audio cue playback tool. You build a list of cues, each pointing at an audio file, then fire them in order with a single key press. Designed to feel familiar to QLab users — keystrokes, terminology, and OSC API match.

Quick Start

  1. Add cues — click + Add Cue in the toolbar (or drag audio files onto the cue list). Files become rows in the cue list.
  2. Select a cue — click its row. The inspector at the bottom shows its properties.
  3. Trim & tune — use the Time tab to set in/out points and listen to a preview, the Audio tab for volume/pan/output device.
  4. Press GO — Space fires the selected cue and advances the playhead to the next.
  5. Save — Cmd S. Projects use the .flowcast format.

Transport & Hotkeys

Continue Modes

Each cue can chain into the next automatically. Set this in the inspector under Continue Mode.

  • Do Nothing — the default. Space must be pressed for each cue.
  • Play With (↷) — the next cue starts at the same time as this one starts. Both play in parallel. Use for layered events: a music bed plus a voiceover firing together. The dotted blue line on the now-playing waveform marks when the linked cue's audio comes in.
  • Auto-Play Next (↪) — the next cue starts when this one finishes. Sequential, no overlap. Use for back-to-back playback.

Pre-wait delays a cue's start after GO — useful for a count-in or external sync. Post-wait adds a delay before the auto-fire trigger launches the next cue. Both fields are in the inspector and apply only when relevant.

Trim, Fades, and Duck

  • Time tab — drag the white markers on the waveform to set in/out points, or type values directly. The Reset button next to Duration restores both to the full file. Click anywhere on the waveform to place the playhead. Space auditions from the playhead; Enter previews the trimmed region.
  • Fade In / Fade Out — applied as smooth ramps at cue start/end. Specified in seconds.
  • Duck — when this cue plays, all other currently-playing cues are smoothly attenuated. Configure the amount, fade-in, and fade-out in the Audio tab. Useful for voiceovers over a music bed.

Combo Cues (Linear Timeline Editor)

Multiple clips bundled into a single cue with a fixed timeline.

  1. Tick the MIX checkbox on two or more cues.
  2. Click BEGIN EDIT in the toolbar to open the LTE.
  3. Drag clips horizontally to position them; drag the white edge handles to trim. Hold Ctrl + scroll to zoom.
  4. Adjust per-clip volume / fade / duck in the right panel.
  5. Click Save to commit. The source cues become a single Combo row in the cue list.

Combo cues fire all their clips at the chosen offsets. Use them for fixed multi-clip moments (intro stinger + bed + voiceover authored as one event). Use Play With when you want the cues to remain independently editable in the cue list instead.

Loop

Tick the Loop checkbox in the inspector. The cue plays seamlessly until stopped. The yellow ↻ icon appears in the cue list. Common use: ambient beds that should run for the duration of a scene.

OSC Remote Control

FlowCast listens on port 53000 by default and speaks the QLab v3/v4 OSC API. Bitfocus Companion's QLab module connects without configuration once you click TCP and point it at your machine.

  • Status dot — top-right of the header. Green = connected, dim = idle, red = error.
  • Click the OSC label to open settings (port, optional passcode, restart).
  • Supported commands: /go, /stop, /pause, /resume, /playhead/next, /playhead/previous, /cue/{N}/start.

Audio Output

Choose your output device in Settings → Output Device. If you plug in or switch interfaces after launch, click the ↻ rescan button to refresh the list. Stop all playback before rescanning.

Live-Show Best Practice

  • Test the whole list in the venue's actual playback setup before doors open. Walk the cues with the same console / Companion buttons you'll use during the show.
  • Set your output device explicitly rather than relying on System Default — the OS can switch defaults when displays connect/disconnect.
  • Number cues with intent — Companion / StreamDeck buttons reference cues by number (/cue/{N}). FlowCast won't auto-renumber on delete or reorder, so externally-mapped buttons stay valid.
  • Prefer Auto-Play Next for back-to-back tracks instead of pressing GO twice in quick succession — keeps timing consistent.
  • Use Duck for voiceovers rather than manually riding faders — less to do live.
  • Save often. The window will warn before closing with unsaved changes, but a crash mid-show isn't covered. Cmd S is your friend.
  • Keep a hot Panic finger — Cmd . kills everything immediately with no fade, for the moment when something has to stop right now.

Troubleshooting

  • No audio — check the output device in Settings; if you connected an interface after launch, click rescan. Verify the cue's volume isn't pulled down.
  • OSC offline — click the status dot, then ↺ Restart. If the port is in use, change it in the popover.
  • Cue won't play / row flashes red — the file failed to load. Check the path; if you moved files, re-add them or fix the path field.
  • Backend exited banner — the Python audio engine crashed. Restart the app. If it persists, the file you tried to play may be malformed.
